# 自定义MCP工具

Perigon内置了一些MCP工具，主要用来生成Asp.Net Core的代码结构。

实际开发中，我们可能需要更多的工具，所以我们通过自定义MCP工具来满足需求。

## 添加自定义工具

在添加自定义工具之前，我们需要先添加[模板](../数据管理/代码模板.md)和[提示词](../数据管理/提示词.md)。然后在`MCP`页面中添加自定义工具。

一个工具，只能选择一个提示词，但可以选择多个模板。

- 提示词：用来告诉工具如何生成代码。
- 模板：用来提供代码结构或参考。

自定义工具根据提示词和模板，生成最终的代码。

添加自定义MCP工具，我们将看到:

![add mcp](../../_images/add_mcp.png)

- 名称：符合mcp规范的名称，即小写加下划线，如`create_service`。
- 说明：非常重要，用来描述该工具的功能，描述准确，以便能够被正确调用。

## 使用自定义工具

在添加自定义工具后，我们需要在客户端重新获取工具列表，才能看到新添加的工具，请打开`mcp.json`文件，点击`Restart`。


![restart_mcp](../../_images/restart_mcp.png)

查看工具列表，自定义工具将出现在列表中，现在你可以在IDE中使用该工具了。
