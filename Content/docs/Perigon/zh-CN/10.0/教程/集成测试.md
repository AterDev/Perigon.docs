# 集成测试

模板默认包含一个集成测试项目，它使用`Tunit`测试框架。位于`test/ApiTest`目录下。

`ApiTest`主要用来测试接口请求和服务间的集成调用。

## 运行测试

要运行集成测试，在测试项目目录下执行以下命令：

```pwsh
dotnet test
```

集成测试会使用`Aspire`来启动所有基础设施和服务，更加接近真实环境。

## 编写测试

你可以借助`AI`来生成测试代码。通常就是请求接口并断言响应结果。

`TestHttpClientData`类包含了`HttpClient`实例，可以用来发送请求。`InitializeAsync`方法有登录获取token的示例，以便在其他接口测试中携带认证信息。
